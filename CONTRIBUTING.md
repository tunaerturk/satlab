# Contributing to satLAB

First off, thank you for considering contributing to satLAB! It's people like you that make satLAB a great tool for students.

## How to Contribute

### Reporting Bugs

- Use the GitHub Issue Tracker.
- Describe the bug and provide steps to reproduce it.

### Suggesting Enhancements

- Open a new issue with a clear description of the enhancement.

### Pull Requests

1. Fork the repository.
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'Add amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request.

## Code Style

- We use [Prettier](https://prettier.io/) for code formatting.
- Run `npm run format` before committing.

## Questions?

Feel free to open an issue for any questions or discussions.
